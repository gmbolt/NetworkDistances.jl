NetworkDistances.get_cost_matrix_dynamic — Method
get_cost_matrix_dynamic(d::AbstractMatchingDistance, X, Y)

Return cost matrix for evaluating matching-based distance between X and Y.

This is done "dynamically" in the sense that the first axis of the output matrix will always correspond to the shorter of the two passed vectors X and Y.

source
NetworkDistances.get_cost_matrix_fixed — Method
get_cost_matrix_fixed(d::AbstractMatchingDistance, X, Y)

Return cost matrix for evaluating matching-based distance between X and Y.

This is "fixed" in the sense the first axis of output matrix always corresponds to X and the second to Y.

StatsAPI.pairwise — Method

Distance matrix calculation between elements of Vectors. This is a custom extension of the function in the Distances.jl package to allow vectors of general type. The function in Distances.jl is designed for univariate/multivariate data and so takes as input either vectors or matrices (data points as rows).
